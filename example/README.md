# Example — Ridgeline, after three jobs

A **fictional** brand (a small outdoor-gear studio), generated by running the growing-brand loop once end-to-end as a test. This is what a `brand/` folder looks like after `/brand-init` plus roughly two weeks of normal work — one landing page, one pricing page, one poster. Nobody sat down to "define the brand."

What to notice, file by file:

- **`journal.md`** — twelve raw judgments, in the user's own words. The marker says entries 1–10 have been distilled; **entries 11–12 are the undistilled tail** — freshest signal, already obeyed, not yet promoted. (The amber "one warm hit per page" line will likely confirm the observed amber token at the next distill.)
- **`tokens.md`** — the init scan seeded the measurables from the project's CSS, all marked `(observed — unconfirmed)`. Two weeks later, some lines carry evidence instead of the marker: *ink, never pure black* was confirmed by two rejections on two different jobs (entries 5, 10); the 4px radius by one correction against an observed line (entry 6).
- **`moves.md`** — patterns, not numbers: photo-led heroes, flat print-like surfaces, dense spec tables. The icons rule went straight in because the user **stated** it outright (entry 8).
- **`core.md`** — the smallest file, as it should be. *"Field, not showroom"* earned its place with three consistent judgments across two jobs (entries 1, 2, 6). *"Real over decorated"* was seeded by an init question and confirmed in use. No tensions yet — when the evidence contradicts itself, it goes here openly instead of being resolved by fiat.

Every promoted rule cites its journal entries. If a rule can't show its evidence, it doesn't get written — that's the whole trick.
